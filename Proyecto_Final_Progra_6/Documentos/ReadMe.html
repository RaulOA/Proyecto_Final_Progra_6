Bienvenido a Claudes 2.0 - Proyecto de inicio ASP.NET Core / Angular

Plantilla de inicio para el desarrollo rápido de aplicaciones.
Esta aplicación proporciona una base conveniente para crear soluciones con ASP.NET Core 9.0 y Angular 19.

Patrocinar

Tecnologías:

  • Páginas plantilla con Angular 19 y TypeScript
  • API RESTful Backend usando ASP.NET Core 9.0 Web API
  • Base de datos con Entity Framework Core
  • Autenticación/Autorización basada en OIDC/OAuth2 y ASP.NET Core Identity
  • Documentación de API usando Swagger
  • Frontend construido con Angular CLI
  • Tematización con Bootstrap 5

Características:

  • Proyecto base simple y listo para desarrollo rápido de aplicaciones
  • Estructura completa backend y frontend con gestión de usuarios y roles basada en permisos
  • Código limpio y sencillo en Angular/ASP.NET Core
  • Capa de acceso a datos con patrón Repositorio y Unidad de Trabajo
  • Base de datos Code First
  • Diseño API RESTful
  • Servicios de Diálogo y Notificaciones
  • Página y servicio de configuración
  • Internacionalización integrada
  • Tematización con SASS
  • API de correo lista para usar

Instalación

  • [OPCIÓN 1] Clona el repositorio en GitHub y edítalo con tu editor favorito (Visual Studio, Visual Studio Code, etc.)
  • [OPCIÓN 2] Descarga el proyecto desde el repositorio oficial y sigue las instrucciones del README para la configuración inicial.

Notas de instalación

  • Al crear un nuevo proyecto, espera a que se restauren todas las dependencias: ejecuta "dotnet restore" para el proyecto ASP.NET y "npm install" para el proyecto Angular.
    Si usas Visual Studio esto es automático, revisa la barra de estado o la ventana de salida para confirmar que la restauración ha finalizado antes de ejecutar la aplicación por primera vez.
  • Si encuentras errores, ejecuta manualmente los pasos de compilación y verifica dónde ocurre el problema.
    Abre la terminal y ejecuta:
    1. 'dotnet restore' en las carpetas de los proyectos - Restaura paquetes NuGet
    2. 'npm install' en el proyecto con package.json - Restaura paquetes npm
    3. Ejecuta la aplicación nuevamente - Verifica que todo funcione correctamente
  • Si ejecutas el cliente (Angular) en una dirección/dominio diferente al backend, configura el baseUrl del cliente para que coincida con el del servidor.
    Esto se realiza en environment.ts del proyecto Angular.
    Ejemplo: baseUrl: "http://tuservidorbackend.com" O baseUrl: "http://localhost:5050"
  • Para ayuda y soporte, abre una incidencia en GitHub
